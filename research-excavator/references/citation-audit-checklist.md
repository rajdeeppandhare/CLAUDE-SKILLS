# Citation Audit Checklist

Used in Stage 8 of the Research Excavator pipeline — the final pass before the report goes out. The point of this stage is narrow and mechanical: every claim that sounds confident needs a receipt. This is the stage that catches the gap between "this is probably true" and "this was actually retrieved."

## The pass

Go through the drafted report claim by claim, not paragraph by paragraph — a single paragraph can mix a well-sourced fact with an unsourced inference, and skimming at the paragraph level is how those slip through.

For each claim, ask:

### 1. Where did this come from?

Point to the specific Stage 3/Stage 6 result that supports it. "I'm pretty sure this is right" is not a source. If you can't name which search result or fetched page a claim traces back to, that's the signal to stop and check, not to keep writing.

### 2. Is this the claim the source actually made?

Sources get paraphrased during synthesis, and paraphrasing drifts. A source saying "growth slowed in Q3" can quietly become "growth declined in Q3" by the time it reaches the draft — these are not the same claim. Re-check the paraphrase against the original, not against your memory of the original.

### 3. Did this get stretched?

Watch specifically for scope-stretching: a source about one company becomes a claim about "the industry," a source about one country becomes "globally," a preliminary finding becomes an established fact, a correlation becomes a cause. None of these are outright fabrication, which is exactly why they're easy to miss — fix the scope back to what the source actually supports.

### 4. Is the number actually in the source?

Specific numbers are the highest-risk claims because they read as maximally confident and are maximally checkable. Any statistic, date, percentage, or dollar figure in the draft should be traceable to an exact spot in a retrieved source. If a number was computed (e.g., a break-even calculation derived from user-provided inputs), label it as a calculation, not a sourced fact.

### 5. Is this from a source that got cut earlier?

Sometimes a source gets used in an early draft, then gets downgraded or dropped in Stage 4/5 (low credibility, contradicted, superseded) — but the claim built on it survives the edit anyway. Cross-check that every cited source in the final draft is still one you'd actually stand behind.

## If a claim fails the audit

Don't soften the wording and leave it in — that's how false precision survives as false hedging instead of getting fixed. Either:

- Go back and find the actual source that supports it, and cite that instead, or
- Cut the claim entirely if nothing retrieved actually supports it, or
- If the claim is plausible but genuinely unverified, say so explicitly in the report ("this wasn't directly confirmed in available sources") rather than presenting it at the same confidence level as verified claims

## A note on what this stage is not

This isn't a copyright/quoting pass — that's a separate, always-on constraint regardless of this skill. This stage is specifically about factual traceability: did this claim actually come from something retrieved, stated accurately, at the right scope, with the right confidence level. A claim can be perfectly paraphrased and still fail this audit if it was never actually in any source to begin with.
